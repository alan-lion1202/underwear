# UI Guidelines

## Purpose

This document keeps the dashboard interface consistent, practical, and suitable for product research and B2B customer development.

## Tables

- Keep essential columns visible by default.
- Use sticky first columns for brand or product names.
- Use column visibility controls for secondary fields.
- Move long fields into detail drawers or expandable rows.
- Support search, filters, tags, status fields, and CSV import/export.

## Cards

- Use cards for summaries, repeated records, and focused information blocks.
- Avoid nesting cards inside cards.
- Keep card radius restrained and practical.

## Dashboard

- Show metrics that support decisions, not vanity counts.
- Prioritize researched brands, data quality, product benchmarks, opportunity scores, and sample candidates.

## Charts

- Use charts for distribution and ranking.
- Keep labels readable.
- Avoid decorative charts that do not support decisions.

## Images

- Brand logos and representative product images should be added only when source rights and URLs are clear.
- Missing images should not block records.

## Navigation

- Keep a clean sidebar layout.
- Use breadcrumb navigation on detail pages.
- Include back buttons on detail pages.

## Responsive Behavior

- Tables should remain readable on desktop.
- Mobile views should collapse layouts without overlapping text.
- Detail views should stack cleanly on small screens.
